# honeybee_grasshopper_ph_plus — HBPH+ Grasshopper plugin (repo root)

Open-source (`github.com/PH-Tools`) plugin: extra utility Grasshopper components for the Honeybee-PH toolchain. Requires the base `honeybee_grasshopper_ph` package. Targets **IronPython 2.7** (Rhino/GH); repo tooling (black/ruff/bumpversion, `.venv`) is CPython. Current version in `pyproject.toml` / `_component_info_.py`.

Two-layer architecture: `honeybee_grasshopper_ph_plus/` is the *GH-facing* package (thin component scripts + compiled user objects + icons); `honeybee_ph_plus_rhino/` is the *backend library* holding the actual component I/O logic that the scripts import.

## Contents

### `honeybee_grasshopper_ph_plus/` — GH plugin package (deployed artifact)
- `src/` — 54 `HBPH+ - *.py` component scripts (thin GH wrappers, one per component). Filenames are self-describing (Airtable, Reporting, SQL, PDF geometry, PHPP readers, window/construction builders, etc.).
- `user_objects/` — 54 matching `.ghuser` compiled Grasshopper user objects (build output; regenerated on release).
- `icons/` — component icons: `AI/` (7 source Illustrator) + `PNG/` (7 exported).

### `honeybee_ph_plus_rhino/` — backend Python library (component logic)
- `_component_info_.py` — component registry: `RELEASE_VERSION`, `CATEGORY` (`HB-PH+`), `SUB_CATEGORIES` map, and per-component `COMPONENT_PARAMS` (NickName/Category/SubCategory). Bumpversion target.
- `gh_compo_io/` — per-component I/O classes, grouped by subcategory: `airtable/` (5), `collections/` (5), `ghpy/` (4), `hb_tools/` (26), `ph_navigator/` (4), `read/` (6), `reporting/` (13); plus `run_subprocess.py`.
- `phpp/` — PHPP data readers (climate, room ventilation, variants); `phpp/bt_web/`.
- `plotly/` — Ladybug data plotting. `sql/` — E+ SQLite readers (column data/names, table names, report variable names).

## Key Files
- `pyproject.toml` — bumpversion config (edits version string in `_component_info_.py`), black/ruff (line-length 120), Py2.7-compat lint ignores.
- `.github/workflows/release.yml` — release automation.
- `typings/` — stub packages (`Grasshopper/`, `System/`) for static typing against the .NET/GH API.
- `README.md`, `LICENSE`, `uv.lock`.

## Notes

Read it before adding/editing a component. Each `gh_compo_io/` subfolder also has its own `.index.md`.
