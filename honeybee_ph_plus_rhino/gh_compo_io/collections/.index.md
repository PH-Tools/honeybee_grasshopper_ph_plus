# gh_compo_io/collections — generic keyed-container components

Canvas subcategory **01 | Collections**. A small set of components for building and
querying a generic keyed collection (dict-like container) on the canvas, plus a
CSV loader. Useful for carrying named data between components without wiring up a
dozen parallel lists.

These are **pure Python** — no Rhino/.NET or network dependency — so they're the
easiest ones to reason about and to test under CPython. Read the parent
`gh_compo_io/.index.md` for the shared facade/IronPython-2.7 conventions.

## Files

- `create_new_collection.py` — `CustomCollection` (the dict-like container type; use
  `CustomCollection.from_dict({key: obj})` to build one from a mapping) + its canvas
  component `GHCompo_CreateCustomCollection` (build a keyed collection from keys + items).
- `create_items_from_csv.py` — `GHCompo_CreateObjectsFromCSV` — parse a CSV into collection items.
- `get_item_from_collection.py` — `GHCompo_GetFromCustomCollection` — look up item(s) by key.
- `set_item_in_collection.py` — `GHCompo_SetInCustomCollection` — set/replace an item by key.
