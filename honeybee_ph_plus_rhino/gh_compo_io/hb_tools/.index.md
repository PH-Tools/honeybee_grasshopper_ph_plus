# gh_compo_io/hb_tools — the general-purpose HB/geometry toolkit

Canvas subcategory **03 | HB-Tools** (plus the SQL readers, see below). This is the
largest and most heterogeneous folder — the grab-bag of utility components that
don't belong to a tighter group. Roughly four clusters:

- **Geometry cleanup / query**: `clean_input_breps`, `check_shade_mesh`,
  `diagnose_hb_rooms`, `get_coplanar_face_groups`, `group_connected_faces`,
  `get_faces_by_name`, `get_rooms_by_name`, `generate_polygon2d_from_faces`,
  `merge_lbt_polygon2ds`, `get_brep_subface_mats`.
- **Sorting**: `sort_geom_objs_by_level`, `sort_hb_objects_by_level` (by building
  level/story).
- **Window builders**: `win_create_geom`, `win_create_types`, `win_rebuild_rh_geom`,
  `win_get_design_ph_shading_factors`.
- **Constructions / envelope**: `steel_stud_construction`, `fastener_correction`,
  `assmbly_import_flixo_mats` (import Flixo 2D-simulation materials),
  `infiltration_from_ach`, `convert_unit`.

Because it's a catch-all, don't infer a folder-wide pattern beyond the shared
`GHCompo_*` convention in the parent `gh_compo_io/.index.md`. `convert_unit.py` is
the cleanest reference example of the class shape.

## SQL readers live here too

`sql_get_*` read from an **EnergyPlus SQLite** results file (`.sql`), not from HB
geometry. They need a valid E+ results path and read table/column/report-variable
data out of it. They sit in this folder for historical reasons but are effectively
their own group.
