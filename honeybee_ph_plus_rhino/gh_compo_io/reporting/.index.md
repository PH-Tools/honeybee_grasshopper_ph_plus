# gh_compo_io/reporting — PDF report geometry, annotations & layout

Canvas subcategory **05 | Reporting**. The richest folder after `hb_tools`. These
components build the geometry, annotations, and layout that get composed into
**PDF report pages** (elevations, floor-segment plans, envelope surfaces, thermal
bridges, ERV ducting diagrams), plus Plotly graph output.

See the parent `gh_compo_io/.index.md` for the shared facade/deploy/IronPython-2.7
rules.

## Folder-specific notes

- **Heavy Rhino/.NET dependency.** Unlike the pure-Python folders, these lean on
  `Rhino.Display`, `System.Drawing` (`Color`, `Size`), `System.Guid`, clipping
  planes, and the active `ghdoc` for layout — all reached through `IGH` and guarded
  .NET imports. They only really run inside Rhino.
- **`to_pdf.py` is the shared exporter**, not a component — it holds the functions
  that actually render/export a PDF report page (note the `izip_longest` Py2/3
  fallback at the top). The `build_*` components produce geometry+attributes that
  feed it.
- **`build_pdf_geom_and_attrs.py`** is the general geometry+attribute builder;
  the other `build_*` files are specializations (elevations, floor segments,
  envelope surfaces, thermal bridges, ERV ducts, simplified wireframe).
- **`create_plotly_graph.py`** bridges to the package-level `plotly/` module for
  Ladybug-data charts — a different output path from the PDF pipeline.
