# gh_compo_io/read — PHPP readers via an out-of-process bridge

Components that read data out of a **PHPP** Excel file (climate, room ventilation,
design variants). These are unusual enough to warrant their own note.

See the parent `gh_compo_io/.index.md` for the shared facade/deploy conventions.

## Why these shell out to a subprocess

The logic here does **not** read the PHPP directly from IronPython. Instead it
spawns an external **CPython 3** process to do the work, because:

- Reading PHPP means driving Excel (`xlwings`) and using `pandas` — neither is
  available in Grasshopper's IronPython 2.7.
- On macOS there's an additional wrinkle: a plain `subprocess` can't get the
  permissions to talk to Excel, so the call is routed through a **shell script in
  its own terminal window** as a workaround (see the McNeel forum link cited in
  `read_phpp_data.py`).

So the flow is: GH component (IPy2.7) → `run_subprocess*` helper → shell script →
CPython3 interpreter → script that reads the PHPP and writes a **CSV** → the CSV
path is returned to the canvas, and a second component parses it.

`os.name == "nt"` (Windows) is currently **not implemented** for the shell-script
path — only macOS is wired up.

## Key shared helper

`../run_subprocess.py` (folder root, one level up) provides
`run_subprocess` / `run_subprocess_from_shell` plus `process_stdout` /
`process_stderr` (which pipe the child process's output back to the canvas via
`IGH.remark` / `IGH.error`). All the `read/` components funnel through it.

## Files

- `read_phpp_data.py` — `run(...)` driver: builds the shell command and launches the CPython3 subprocess (no `GHCompo_*` class; it's the engine).
- `read_phpp_data_climate.py` — `GHCompo_ReadPHPPClimateData` — read climate data.
- `read_phpp_data_room_vent.py` — `GHCompo_ReadPHPPRoomVentilationData` — read room-ventilation data.
- `read_phpp_data_variants.py` — `GHCompo_ReadPHPPVariantsData` — read design-variants data.
- `process_phpp_csv_data.py` — `GHCompo_ProcessPHPPCSVData` — parse/clean the CSV the subprocess produced.
