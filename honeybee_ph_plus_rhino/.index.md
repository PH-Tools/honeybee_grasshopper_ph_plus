# honeybee_ph_plus_rhino/ — backend library (component logic)

The logic half of HBPH+: the `GHCompo_*` classes and the integration libraries they use. See `../context/ARCHITECTURE.md`.

## Contents

- `gh_compo_io/` — the component logic classes, grouped by subcategory (`airtable/`, `collections/`, `ghpy/`, `hb_tools/`, `ph_navigator/`, `read/`, `reporting/`). **Start here** — it has its own `.index.md` explaining the two-layer/fsdeploy design and `GHCompo_*` anatomy, and each subfolder has one too.
- `_component_info_.py` — the registry: `RELEASE_VERSION`, `CATEGORY` (`HB-PH+`), `SUB_CATEGORIES`, `COMPONENT_PARAMS`. bump-my-version target; a new/renamed component needs an entry.
- `phpp/` — PHPP data readers (climate, room ventilation, variants). See `phpp/.index.md`.
- `plotly/` — Ladybug data plotting. See `plotly/.index.md`.
- `sql/` — EnergyPlus SQLite readers. See `sql/.index.md`.

## Notes
- IronPython-2.7-safe; deployed to Rhino via fsdeploy on save.
